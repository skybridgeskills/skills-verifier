# Phase 3: Create Example Card Component

## Objective
Create a presentational Card component with spacing, responsiveness, and visual wrapping using Tailwind classes.

## Tasks
- Create `src/lib/components/Card.svelte`
- Implement presentational component with:
  - Proper spacing (padding, margins)
  - Responsive design (container queries)
  - Visual wrapping (border, shadow, rounded corners)
  - Slot for content
- Use Tailwind classes for all styling
- Ensure component follows SvelteKit conventions

## Component Requirements
- Presentational only (no behavior/interactivity)
- Responsive using container queries (`@md:`, `@lg:`, etc.)
- Proper spacing and visual hierarchy
- Accessible markup

## Files to Create
- `src/lib/components/Card.svelte`

## Success Criteria
- Card component created with proper structure
- Uses Tailwind classes for styling
- Responsive design implemented
- Component is accessible
- Code compiles without errors
- Component can be imported and used

## Tests
- Verify component file exists
- Check that component compiles without errors
- Verify Tailwind classes are applied correctly
- Test component renders properly when imported
